redp is the release deployment platform built for Nova release managers. It brings together your release scope, component risk data, Jira and GitLab evidence, CMR strategy, and rollback analysis into a single auditable workflow — replacing ad-hoc spreadsheets and manual checklist reviews with structured guideline reports and deterministic approval decisions that every stakeholder can trust.

## Approval decision model

Every guideline report redp generates closes with one of four deterministic approval decisions. The decision is computed by evaluating all readiness rules against the collected evidence — there is no manual override of the outcome itself, only the ability to file a waiver or resolve a blocker.

| Decision | Meaning |
|---|---|
| **GO** | All rules pass. The release is ready to proceed. |
| **CONDITIONAL\_GO** | One or more rules are conditional. Proceed only after resolving or waiving each condition. |
| **NO\_GO** | One or more rules are blocking. The release must not proceed until blockers are resolved. |
| **UNKNOWN** | Evidence collection is incomplete or the report is still processing. Re-evaluate before making a release decision. |

<Tip>
  A CONDITIONAL\_GO is not a failure — it means the release can move forward once the listed conditions are acknowledged and signed off by the relevant owners.
</Tip>
